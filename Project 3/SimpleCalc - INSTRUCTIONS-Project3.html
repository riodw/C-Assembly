CompSci 240 parse-arithmetic assignment

CompSci 240, Low-level Programming

assignment: Parse Arithmetic Expression

points?

Overview

Write a program that reads simple arithmetic expressions from stdin, and evaluates them. (This is an exercise in if and switch statements.)

Legal expressions have the form: 


    <left-value>  <operator>  <right-value>
where <left-value> and <right-value> are floating-point numbers, and <operator> is a single-character operator such as +, -, *, /, etc. The full list of required operators is given below. The operator may be surrounded by whitespace, but doesn't have to be. You should be able to handle expressions with whitespace around the operator, as well as expressions without whitespace (see the example output below).

You may assume that inputs are syntactically legal (the user follows the pattern correctly), and the numbers are legally formed values. However, it must handle improper/unrecognized operators with a suitable message. Also, some legal inputs do not give valid arithmetic. For example, division by 0 is illegal. Your program must check for such special cases, and print out a suitable message instead of trying to do the math.

Turn in:

Submit
  • the source-code file
  • a file containing a sample run, similar to the one below

Source code files must begin with a comment containing your name, the name of the assignment and a brief description of it (one line is sufficient), and the date you wrote it. For example: 

/*
  Alice ByteCoder
  Assignment 1: Program to show powers of numbers in a table
  October 15, 1582
/*

Grading

  • 30%:  program compiles and runs.
  • 50%:  program produces correct answers for all operators, over entire domain of inputs
  • 20%:  program output is nicely formatted.

Required operators:

  • + — addition of floats
  • - — subtraction of floats
  • * — multiplication of floats
  • / — division of floats
  • \ — division of integers
  • % — modulo (remainder) of integers
  • p — power: left-arg raised to the power of right-arg
  • l — logarithm: log( left-arg ) to the base right-arg
